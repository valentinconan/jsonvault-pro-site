JSONVault Pro EN

Politique de confidentialité

Dernière mise à jour : 27 février 2026

En résumé : JSONVault Pro traite les données localement sur votre appareil. Nous ne collectons pas, ne transmettons pas et ne vendons pas vos données personnelles ni le contenu des documents que vous consultez.

1. Qui sommes-nous

JSONVault Pro est une extension Chrome développée et maintenue en tant que projet indépendant. Pour toute question relative à la vie privée, contactez-nous à support@jsonvaultpro.com.

2. Données que nous ne collectons PAS

JSONVault Pro ne collecte pas, ne stocke pas sur des serveurs externes et ne transmet pas :

  • Le contenu des documents JSON, YAML ou XML que vous consultez
  • Les URL que vous visitez
  • Toute information personnellement identifiable
  • L'historique de navigation ou les données d'analyse comportementale

3. Données stockées localement sur votre appareil

JSONVault Pro stocke les données suivantes exclusivement dans le stockage local de votre navigateur (chrome.storage.local), qui ne quitte jamais votre appareil :

  • Préférences utilisateur : thème, mode d'affichage, taille de police, fonctionnalités activées et autres paramètres d'affichage.
  • Historique des documents (Pro) : les 50 derniers documents consultés, conservés pour votre commodité. Vous pouvez effacer cet historique à tout moment depuis la page Historique.
  • Statut de licence : si votre compte dispose d'un abonnement Pro actif (mis en cache localement pour éviter des appels réseau répétés).

4. Vérification de licence (niveau Pro)

Si vous achetez une licence Pro, l'extension communique avec ExtensionPay uniquement pour vérifier votre statut de licence. Cela implique :

  • L'envoi de votre jeton d'identité Chrome aux serveurs d'ExtensionPay pour vérifier le statut de l'abonnement.
  • Aucun contenu de document ni donnée de navigation n'est inclus dans cette requête.

La politique de confidentialité d'ExtensionPay s'applique à cette interaction : extensionpay.com/privacy.

5. Permissions utilisées

L'extension demande les permissions Chrome suivantes, chacune avec un objectif spécifique et limité :

  • activeTab / scripting : pour injecter l'interface de visualisation dans l'onglet courant lorsqu'une URL JSON/YAML/XML est détectée.
  • storage : pour sauvegarder vos préférences et votre historique localement.
  • alarms : pour planifier le nettoyage périodique des données expirées (TTL de l'historique).
  • devtools : pour fournir le panneau DevTools permettant d'inspecter les réponses réseau.

6. Services tiers

JSONVault Pro n'intègre aucun réseau publicitaire, plateforme d'analyse ni courtier en données. Le seul service tiers impliqué est ExtensionPay, utilisé exclusivement pour la gestion des licences Pro (voir section 4).

7. Modifications de cette politique

Nous pouvons mettre à jour cette politique de confidentialité au fil de l'évolution de l'extension. Les modifications seront reflétées par la mise à jour de la date « Dernière mise à jour » en haut de cette page. Les changements importants seront communiqués via la bannière « Nouveautés » de l'extension.

8. Contact

Pour toute question concernant cette politique de confidentialité, contactez-nous à support@jsonvaultpro.com ou ouvrez un ticket sur GitHub.
